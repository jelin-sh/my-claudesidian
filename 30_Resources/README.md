# 📚 资源（Resources）

你的参考材料和常青笔记的个人知识库。

## 用途

资源是**参考资料**，它们：
- 你将来可能需要
- 不与特定项目绑定
- 代表持续感兴趣的主题
- 随时间建立你的知识库

## 什么属于这里

### 考虑的分类
```
30_Resources/
├── Articles/          # 保存的网页内容
├── Books/            # 书籍笔记和摘要
├── Concepts/         # 常青想法笔记
├── People/           # 杰出人物和思想家
├── Tools/            # 软件、方法、框架
├── Quotes/           # 难忘的引用
├── Examples/         # 案例研究、参考
└── Learning/         # 课程笔记、教程
```

### 好的资源
- Python 装饰器教程
- 心智模型文章
- 《思考，快与慢》读书笔记
- 决策框架
- 认知偏差列表
- 行业趋势分析

## 组织原则

### 按主题，而非来源
❌ "来自 Medium 的文章"
✅ "AI 开发技术"

### 常青优于短暂
❌ "2024 年 3 月新闻"
✅ "网络效应原则"

### 原子笔记
- 每个笔记一个概念
- 自包含的理解
- 与相关想法大量链接

## Claude Code 工作流

### 建立知识
```
我正在研究 [主题]。
我在 Resources 中已经有什么？
我的知识中存在什么空白？
```

### 建立联系
```
找到与 [概念] 相关的所有资源。
不同来源如何处理这个问题？
出现什么模式？
```

### 创建综合
```
查看 Resources 中关于 [主题] 的所有笔记。
创建关键见解的综合文档。
```

### 寻找示例
```
我需要 [概念] 的示例。
在我的资源中搜索相关案例。
```

## 资源开发

### 从消费到创造
1. **捕捉**：保存有趣的内容
2. **处理**：提取关键想法
3. **连接**：链接到现有笔记
4. **发展**：建立你自己的理解
5. **创造**：产生原创见解

### 渐进式摘要
- 第一遍：高亮有趣的部分
- 第二遍：加粗最重要的内容
- 第三遍：在顶部创建摘要
- 第四遍：提取到自己的笔记

## Claude Code 提示

### 知识审计
```
分析我的 Resources 文件夹。
我在哪些主题上正在建立专业知识？
空白在哪里？
```

### 连接发现
```
在我的 Resources 中不同主题之间找到惊人的联系。
```

### 学习路径
```
基于我关于 [主题] 的资源，
我下一步应该学习什么？
创建学习路径。
```

### 概念澄清
```
使用我现有资源中的示例解释 [概念]。
```

## 最佳实践

### 命名约定
- 清晰、描述性的标题
- 在名称中包含关键概念
- 避免日期（除非是历史性的）
- 使用一致的格式

### 笔记结构
```markdown
# 概念名称

## 摘要
一段概述

## 关键点
- 主要想法 1
- 主要想法 2
- 主要想法 3

## 连接
- 相关于：[[其他概念]]
- 对比于：[[不同想法]]
- 示例：[[案例研究]]

## 来源
- 原始文章/书籍/视频
```

### 维护
- 定期审查和更新
- 合并重复的概念
- 加强连接
- 删除不再相关的内容

## 提示

- **质量优于数量** - 深入理解少数概念更好
- **你的语言很重要** - 用你自己的理解重写
- **链接是金** - 连接创造价值
- **定期回顾** - 未使用的知识会消退
- **自由分享** - 教学巩固理解

## 记住

资源是你的外部大脑。价值不在于收集，而在于连接和创造。使用 Claude Code 帮助你看到模式并在整个知识库中建立理解。
