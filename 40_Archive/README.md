# 🗄️ 归档（Archive）

为未来参考保存的非活动项目。

## 用途

归档存储：

- 已完成的项目及其输出
- 不再维护的不活动领域
- 供历史参考的旧笔记
- 已弃用的资源
- 过去的实验和迭代

## 什么放在这里

### 来自 Projects

- 具有最终可交付成果的已完成项目
- 具有经验教训的已取消项目
- 30+ 天未活动的项目

### 来自 Areas

- 与你生活不再相关的领域
- 你已经移交的责任
- 你不再拥有的角色

### 来自 Resources

- 过时信息（但历史上有趣）
- 被取代的框架或方法
- 演进想法的旧版本

### 来自 Inbox

- 已处理且不再需要的项目
- 旧每日笔记（提取后）
- 没有持久价值的随机捕捉

## 组织

```
40_Archive/
├── Projects_2024/     # 按年份完成的已完项目
├── Projects_2023/
├── Old_Areas/        # 已停止的领域
├── Daily_Notes/      # 旧的每日捕捉
├── Ideas/           # 未发展的想法
└── Miscellaneous/   # 其他所有内容
```

## 归档过程

### 归档项目前

1. 创建完成摘要
2. 将可重用的见解提取到 Resources
3. 记录经验教训
4. 更新任何相关的 Areas
5. 完整移动文件夹及其结构

### 完成摘要示例

```markdown
# 项目：[名称] - 完成摘要

**持续时间**：开始日期 - 结束日期 **状态**：已完成/已取消/已暂停

## 目标

- 原始目标 1 ✓
- 原始目标 2 ✓
- 原始目标 3 ✗

## 关键结果

- 交付了什么
- 有什么影响
- 创造了什么价值

## 经验教训

- 什么运作良好
- 什么不工作
- 下次不同的做法

## 可重用资产

- 创建的模板：[[link]]
- 开发的流程：[[link]]
- 获得的见解：[[link]]

## 相关笔记

- 继续于：[[领域名称]]
- 另见：[[相关项目]]
```

## 归档哲学

### 不是墓地

- 归档保留机构记忆
- 旧项目启发新项目
- 模式随时间出现
- 想法可以复活

### 不是垃圾场

- 有思想地归档
- 保持一定组织
- 保持摘要可访问
- 删除真正的垃圾

## Claude Code 提示

### 历史分析

```
查看我的已归档项目。
我倾向于开始但没完成什么类型的事情？
什么主题反复出现？
```

### 知识挖掘

```
在归档中搜索任何提到 [概念] 的内容。
我的思维如何演进？
```

### 模式识别

```
分析我的项目完成率。
什么因素与成功相关？
什么模式预测失败？
```

## 维护

### 每季度

- 审查最近的添加
- 确保摘要存在
- 检查复活候选者

### 每年

- 重大归档清理
- 删除真正死的内容
- 提取任何遗漏的见解
- 必要时重新组织

## 提示

- **给所有内容加上日期** - 未来的你会感谢你
- **总是摘要** - 上下文很快消退
- **自由链接** - 连接在归档中幸存
- **经常搜索** - 归档旨在被使用
- **无畏删除** - 不是所有内容都需要保留

## 记住

归档是你的机构记忆。它不是关于保留所有内容，而是关于保存可能告知未来工作的内容。使用 Claude Code 帮助你看到跨时间的模式并从经验中提取智慧。
