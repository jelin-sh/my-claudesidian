# 每周综合

创建一周工作和思考的综合总结。

## 分析流程

1. **收集一周工作**
   - 本周创建的所有笔记
   - 本周修改的所有笔记
   - 有活动的项目

2. **识别模式**
   - 反复出现的主题
   - 常见挑战
   - 突破时刻
   - 能量模式（什么让人精力充沛 vs 消耗精力）

3. **综合学习**
   - 出现的关键洞察
   - 思维如何演变
   - 发现的连接
   - 已回答和提出的问题

4. **评估进展**
   - 推进的项目
   - 维护的领域
   - 添加的资源
   - 归档的项目

## 输出格式

创建每周综合笔记：

```markdown
# 每周综合 - [日期]周

## 一览

- 创建笔记：[X]
- 活跃项目：[列表]
- 主要成就：[列表]

## 关键主题

### 主题 1：[名称]

- 出现场合：[上下文]
- 为何重要：[意义]
- 后续行动：[做什么]

### 主题 2：[名称]

- 出现场合：[上下文]
- 为何重要：[意义]
- 后续行动：[做什么]

## 主要洞察

1. [有上下文的洞察]
2. [有上下文的洞察]

## 按项目进展

### [项目名称]

- 推进内容：
- 受阻内容：
- 下周重点：

## 出现的问题

- [问题 1 - 及为何重要]
- [问题 2 - 及为何重要]

## 能量审计

- 给予能量的事：
- 消耗能量的事：
- 需要调整的事：

## 建立的连接

- [笔记 A] ←→ [笔记 B]：[为何重要]
- [概念 X] ←→ [概念 Y]：[新理解]

## 下周意图

1. [主要重点]
2. [次要重点]
3. [要探索的事]

## 待处理

- 收件箱项目：[数量]
- 孤立笔记：[列表]
- 缺失的连接：[已识别]
```

## 后续行动

- 归档已完成的项目
- 清理收件箱
- 更新项目状态
- 规划下周重点
