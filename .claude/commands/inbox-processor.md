# 收件箱处理器

帮助你按照 PARA 方法组织和处理 00_Inbox 文件夹中的项目。

## 任务

审阅 `00_Inbox/` 中的所有笔记并帮助分类：

1. **扫描收件箱**
   - 列出当前 00_Inbox 中的所有文件
   - 排除 README.md 和 Welcome.md

2. **分析每个项目**
   - 阅读内容
   - 识别笔记类型
   - 建议适当的目标位置

3. **分类规则**
   - **→ 10_Projects**：有截止日期、具体结果
   - **→ 20_Areas**：持续责任、无结束日期
   - **→ 30_Resources**：参考资料、知识
   - **→ 40_Archive**：旧的/已完成、不再活跃
   - **→ 删除**：无价值、冗余或临时内容

4. **建议操作**

   ```
   文件：[文件名]
   类型：[检测到的类型]
   目标位置：[建议的文件夹]
   原因：[为何这样分类]
   关联到：[它连接的任何现有笔记]
   ```

5. **识别模式**
   - 多个笔记中的共同主题
   - 可以合并的笔记
   - 项目之间缺失的连接

## 输出格式

提供清晰的行动计划：

1. 要移动的项目（附带目标位置）
2. 要合并或链接的项目
3. 要删除的项目
4. 需要更多背景的项目

## 记住

- 有些项目确实属于收件箱（每日笔记、快速捕捉）
- 不要过度组织 - 有时"足够好"就是完美
- 寻找连接想法的机会，而不仅仅是归档它们
