# 📥 收件箱（Inbox）

你的新想法、快速思考和未处理信息的捕捉区域。

## 用途

收件箱是一个**临时**存放区域，专为以下目的设计：
- 快速捕捉而无需担心组织
- 每日笔记和日记
- 网页剪藏和有趣的发现
- 处理前的会议记录
- 随机的想法和灵感

## 如何使用

### 每日工作流
1. **先捕捉，后整理** - 不要为了分类而放慢速度
2. **使用每日笔记** - 每天一个笔记，记录意识流
3. **倾倒链接和引用** - 现在保存，稍后处理
4. **不要堆积** - 每周处理一次

### 每周处理
每周回顾你的收件箱并：
- 将项目相关的笔记移到 `10_Projects/`
- 将持续进行的话题移到 `20_Areas/`
- 将参考资料移到 `30_Resources/`
- 将已完成项目归档到 `40_Archive/`
- 删除不再相关的内容

## Claude Code 命令

### 快速捕捉
```
在 00_Inbox 中创建一个名为 [标题] 的新笔记
包含这些想法：[内容]
```

### 处理收件箱
```
查看 00_Inbox 中的所有笔记。
根据 PARA 方法帮我决定每个笔记应该放哪里。
```

### 发现联系
```
查看收件箱中的近期笔记。
你看到什么模式或主题？
这些笔记可能与现有笔记有什么联系？
```

## 提示

- **不要追求完美** - 收件箱本来就可以有些乱
- **使用描述性文件名** - 让后续处理更简单
- **给笔记加上日期** - YYYY-MM-DD 格式便于排序
- **自由建立链接** - 即使链接的笔记还不存在
- **定期回顾** - 不要让条目停留超过 2 周

## 文件命名

建议格式：
- 每日笔记：`2024-03-15.md`
- 快速捕捉：`2024-03-15 - 团队会议.md`
- 网页剪藏：`2024-03-15 - 文章标题 - 来源.md`
- 想法：`想法 - 简短描述.md`

记住：收件箱是想法开始的地方，不是它们永远停留的地方。
