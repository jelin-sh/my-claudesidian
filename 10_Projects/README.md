# 🎯 项目（Projects）

具有明确成果和截止日期的主动倡议。

## 用途

项目是**有时间限制的努力**，具有：
- 明确的目标和可交付成果
- 定义的开始和结束日期
- 具体的成功标准
- 可衡量的结果

## 什么属于这里

### 好的项目
- "2024 年 Q1 市场策略" ✓
- "网站重新设计" ✓
- "书籍写作 - AI 与创造力" ✓
- "会议演讲 - BRXND NYC" ✓

### 不是项目（这些放在 Areas）
- "健康" ✗（持续进行，无结束日期）
- "学习" ✗（持续活动）
- "团队管理" ✗（持续责任）

## 项目结构

每个项目应该有：
```
项目名称/
├── README.md           # 项目概览和状态
├── Research/          # 背景材料
├── Daily_Progress/    # 工作会话日志
├── Drafts/           # 进行中的工作
├── Resources/        # 支持文档
└── Output/          # 最终可交付成果
```

## Claude Code 工作流

### 启动项目
```
在 10_Projects 中创建一个名为 [名称] 的新项目。
使用标准文件夹结构进行设置。
我处于思考模式 - 帮助我定义目标。
```

### 项目研究
```
我正在做 [项目名称]。
在我的 vault 中搜索相关的现有笔记。
与我其他工作有什么联系？
```

### 每日进度
```
在 [project]/Daily_Progress 中为今天创建进度笔记。
这是我完成的内容：[摘要]
出现的问题：[列表]
```

### 项目状态
```
查看 [project folder] 中的所有笔记。
当前状态是什么？
到目前为止的关键见解是什么？
还有什么未完成？
```

### 项目完成
```
项目 [name] 已完成。
创建一个回顾，涵盖：
- 目标与结果
- 关键学习
- 下次不同的做法
然后帮我正确归档它。
```

## 项目生命周期

1. **启动**：定义目标、成功标准
2. **研究**：收集相关信息
3. **开发**：每日进度、迭代工作
4. **审查**：定期状态检查
5. **完成**：最终输出、回顾
6. **归档**：移到 `40_Archive/` 并附摘要

## 成功提示

- **从明确的目标开始** - 模糊的项目永远不会结束
- **使用模板** - 一致性有助于 Claude Code 帮助你
- **每天记录进度** - 即使"没有进度"也值得记录
- **每周回顾** - 保持项目正轨
- **完成或终止** - 不要让项目拖延

## 何时归档

将项目移到 `40_Archive/` 当：
- 所有目标已完成
- 项目被取消
- 已 30+ 天未活动
- 已转变为 Area（持续进行）

归档前务必创建摘要笔记！
